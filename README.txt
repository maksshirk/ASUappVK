УСТАНОВКА xTUNNEL ДЛЯ WINDOWS

Рекомендуемый способ установки — через Chocolatey:
https://xtunnel.ru/docs/install/windows/chocolatey

Он обеспечивает автоматическую установку, настройку и обновление утилиты.

Однако, если вы скачали архив вручную, следуйте данной инструкции.


КРАТКАЯ ИНСТРУКЦИЯ
	1.	Установите сертификат: дважды щёлкните файл xtunnel-cert.cer и добавьте его в хранилище доверенных издателей.
	2.	Откройте PowerShell, перейдите в папку с xtunnel.exe и выполните:
        .\xtunnel.exe --version
	3.	При первом запуске активируйте:
        .\xtunnel.exe register ВАШ_КЛЮЧ
	4.	Запустите туннель (HTTP или TCP):
        .\xtunnel.exe http 80
        .\xtunnel.exe tcp 5432
	5.	(Опционально) Добавьте путь к утилите в переменные среды PATH.

ПОДРОБНАЯ ИНСТРУКЦИЯ

    1.	Установите сертификат издателя
            Чтобы избежать блокировок со стороны Windows, установите сертификат:

        •	Дважды щёлкните файл xtunnel-cert.cer
        •	Нажмите «Установить сертификат»
        •	Выберите «Текущий пользователь»
        •	Укажите: «Поместить в хранилище» → «Обзор»
        •	Выберите хранилище: «Доверенные издатели» или «Доверенные корневые центры сертификации»
        •	Завершите установку, подтвердив действия

        Если вы уже устанавливали сертификат ранее — этот шаг можно пропустить.

    2.	Проверьте работу утилиты

        •	Откройте новое окно PowerShell
        •	Перейдите в папку, где находится xtunnel.exe
        •	Выполните команду:
            .\xtunnel.exe --version

        Если всё установлено корректно, вы увидите номер текущей версии (например, 2.0.0).

    3.	Активация xTunnel
        Если вы запускаете xTunnel впервые, понадобится ввести ключ активации:
        .\xtunnel.exe register ВАШ_КЛЮЧ

        Где ВАШ_КЛЮЧ — ключ активации (FREE, PRO, BUSINESS или ENTERPRISE).

    4.	Запуск туннеля

        Для HTTP туннеля (например, на порт 80):
        .\xtunnel.exe http 80

        Для TCP туннеля (например, на порт 5432 для PostgreSQL):
        .\xtunnel.exe tcp 5432

        Если всё прошло успешно, появится публичный URL или порт для подключения.

    5.	(Опционально) Добавьте путь в переменные среды
        Чтобы запускать xTunnel из любой директории:

        •	Переместите xtunnel.exe (и, при желании, xtunnel-cert.cer) в постоянную папку, например:
            C:\Programs\xTunnel\
        •	Откройте: Пуск → "Система" → "Дополнительные параметры системы"
        •	Нажмите «Переменные среды…»
        •	В разделе «Переменные среды пользователя» найдите переменную Path, нажмите «Изменить»
        •	Нажмите «Создать» и добавьте путь:
            C:\Programs\xTunnel\
        •	Подтвердите изменения и перезапустите PowerShell

        Теперь вы можете запускать утилиту из любой директории:
        xtunnel --version
        xtunnel http 80
        xtunnel tcp 5432

    6.	Обновление
        Для обновления утилиты вручную скачайте новую версию с сайта https://xtunnel.ru и замените файл xtunnel.exe
